개념 — 웹앱 3종 세트概念 — 网页三件套

코드 = 텍스트, 웹앱은 3종 세트代码 = 文本，网页应用是三件套

우리가 만들 웹앱은 세 가지 언어가 합쳐져 하나의 화면이 돼요. 각자 하는 일이 달라요 — 사람으로 치면 뼈대·옷·움직임이에요.我们要做的网页应用，由 三种语言 合成一个画面。各司其职 —— 用人来比就是 骨架·衣服·动作。

  • HTML — 뼈대 · 무엇이 있는지 (제목, 버튼, 이미지)骨架 · 有什么(标题、按钮、图片)
  • CSS — 꾸미기 · 어떻게 보이는지 (색, 크기, 배치)美化 · 长什么样(颜色、大小、排版)
  • JavaScript — 움직임 · 무엇을 하는지 (클릭, 반응, 계산)交互 · 会做什么(点击、反应、计算)
✍️ 코드는 ‘텍스트’라서 편집기(VS Code)를 써요代码是「文本」，所以用编辑器(VS Code)

HTML·CSS·JS 모두 결국 글자(텍스트)예요. 그래서 사실 메모장으로도 만들 수 있어요. 하지만 VS Code는 문법을 색으로 구분해 주고 자동완성·오류 표시·미리보기로 코드 쓰기를 훨씬 쉽게 해줘요 — 그래서 제일 먼저 VS Code를 깝니다.HTML·CSS·JS 归根到底都是 文字(文本)，所以其实 用记事本也能 写。但 VS Code 会给语法 上色，还有 自动补全·错误提示·预览，让写代码轻松得多 —— 所以先装 VS Code。

개념 — GUI vs CLI概念 — GUI vs CLI

검은 창(CLI)이 뭐예요?黑窗口(CLI)是什么?

이제부터 도구들을 검은 창에 글자를 쳐서 설치할 거예요 — 이게 바로 CLI예요. 평소 쓰던 클릭 방식(GUI)과 뭐가 다른지 알면 하나도 안 무서워요.接下来我们会 在黑窗口里打字 来安装工具 —— 这就是 CLI。搞懂它和平时的 点击方式(GUI) 有什么不同，就一点也不可怕。

🖱️ GUI — 클릭 방식点击式

아이콘·버튼을 눈으로 보고 클릭. 평소 쓰는 화면.看着图标·按钮 用眼看、用手点。平时用的界面。

⌨️ CLI — 명령어 방식命令式

검은 창에 글자로 명령을 타이핑. 빠르고 정확.在黑窗口里 用文字敲命令。快而准。

🔎 더 깊이 — 개발자들이 CLI를 좋아하는 이유深入了解 — 开发者为什么爱用 CLI
  • 빠르고 정확 — 메뉴를 찾아 클릭할 필요 없이 명령 한 줄로 끝. 복사·붙여넣기하면 오타도 없어요.快而准 —— 不用翻菜单，一行命令搞定，复制粘贴还不会打错。
  • 자동화·반복 — 여러 작업을 하나로 묶어 한 번에 실행 (수백 번도 똑같이).自动化·重复 —— 把多步打包一次执行(几百次也一样)。
  • 어디서나 똑같이 — 화면 없는 서버·클라우드도 명령어로 다뤄요.到哪都一样 —— 没有画面的 服务器·云 也用命令操作。
  • 기록·공유가 쉬움 — 명령을 그대로 복사해 남기고 나눌 수 있어요 (이 자습서의 코드 상자처럼!).易记录·分享 —— 命令可原样复制、保存、分享(就像本手册的代码框!)。
개념 — 검은 창 열기概念 — 打开黑窗口

우리가 쓸 검은 창 — PowerShell我们用的黑窗口 — PowerShell

💡 우리가 쓸 검은 창我们要用的黑窗口

윈도우에선 PowerShell, VS Code 안에선 터미널(Terminal)이 바로 그 CLI예요. 겁먹지 말고 복사 → 붙여넣기 → Enter만 하면 돼요. (붙여넣기는 보통 마우스 우클릭 또는 Ctrl+V.)Windows 里是 PowerShell，VS Code 里是 终端(Terminal)，都是 CLI。别怕，只要 复制 → 粘贴 → 回车。(粘贴通常是 右键 或 Ctrl+V。)

설치 — VS Code安装 — VS Code

① VS Code 설치① 安装 VS Code

방금 연 PowerShell에 아래 명령을 복사 → 붙여넣기 → Enter. 이번 장에서 도구를 하나씩 깔아요.在刚打开的 PowerShell 里 复制 → 粘贴 → 回车 下面的命令。这一章逐个安装工具。

📦 winget이 뭐예요?winget 是什么?

명령마다 나오는 winget은 윈도우에 들어 있는 ‘앱 설치기’예요. winget install 프로그램이름 = “이 프로그램 설치해줘”라는 뜻. 그래서 우리는 클릭 없이 명령 한 줄로 도구를 깔 수 있어요.每条命令里的 winget 是 Windows 内置的「应用安装器」。winget install 程序名 = 「帮我装这个程序」。所以不用点击，一行命令就能装工具。

PowerShell
winget install Microsoft.VisualStudioCode
⚠️ “Add to PATH” 꼭 체크务必勾选 “Add to PATH”

설치 화면에서 “Add to PATH”가 보이면 체크하세요 — 그래야 나중에 터미널에서 code 명령이 켜져요. (winget이 안 되면 code.visualstudio.com에서 직접 설치.)安装界面看到 “Add to PATH” 就 勾选 —— 之后终端里 code 命令才能用。(winget 不行就到 code.visualstudio.com 手动装。)

체크 화면이 안 보이거나 놓쳤다면? 아래 명령 한 줄로 code를 등록할 수 있어요.没看到勾选界面或漏了? 用下面一行命令注册 code。

PowerShell
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";$env:LocalAppData\Programs\Microsoft VS Code\bin", "User")
설치 — Node安装 — Node

② Node.js 설치② 安装 Node.js

OpenCode는 npm(도구를 한 줄로 까는 ‘앱스토어’)으로 설치해요. 그 npm이 Node.js(브라우저 밖에서도 자바스크립트를 돌리는 엔진) 안에 들어 있어서, AI 도구를 깔려면 Node부터 필요해요.OpenCode 用 npm(一行命令装工具的「应用商店」)安装。而 npm 在 Node.js(让 JavaScript 在浏览器外也能运行的引擎)里 —— 所以装 AI 工具要 先装 Node。

기초 — 컴퓨터基础 — 电脑

컴퓨터는 ‘시키는 것만’ 하는 기계电脑是「只做被吩咐的事」的机器

컴퓨터는 엄청나게 빠르지만, 스스로 알아서 하는 건 하나도 없어요. 사람이 “이걸 이렇게 해”라고 정확히, 순서대로 시켜야만 움직여요. 그 ‘시키는 말’을 컴퓨터가 알아듣는 형태로 적어둔 것이 바로 프로그램이에요.电脑非常快，但 它不会自己做任何判断。 必须由人 精确地、按顺序 吩咐「把这个这样做」，它才会动。把这些「吩咐的话」写成电脑能听懂的样子，就是 程序。

우리가 쓰는 카카오톡, 유튜브, 게임 — 전부 누군가가 미리 적어둔 프로그램이에요. 바이브 코딩은 그 프로그램을, 어려운 문법 대신 AI에게 말로 부탁해서 만드는 방법이고요.我们用的微信、YouTube、游戏 —— 全都是别人事先写好的程序。Vibe 编程 就是不用难懂的语法，而是 用话拜托 AI 来做出这些程序的方法。

🔎 더 깊이 — 컴퓨터는 결국 ‘켜짐/꺼짐’深入了解 — 电脑归根到底是「开/关」

컴퓨터 속은 사실 0과 1(전기가 꺼짐/켜짐) 두 가지만 다뤄요. 우리가 보는 글자·그림·소리는 전부 이 0과 1의 엄청난 조합으로 표현된 거예요. 사람이 0과 1을 직접 쓰긴 너무 힘드니까, 사람 말에 가까운 프로그래밍 언어를 만들어 두고, 그걸 컴퓨터가 이해하는 0·1로 번역해서 돌려요. (이 ‘번역’은 뒤 부록에서 다시 나와요.)电脑内部其实只处理 0 和 1(电的关/开)两种。我们看到的文字·图片·声音，全是这些 0 和 1 的海量组合。人直接写 0 和 1 太难，所以造了更接近人话的 编程语言，再 翻译 成电脑懂的 0·1 来运行。(这个「翻译」在后面附录会再讲。)

기초 — 파일과 폴더基础 — 文件与文件夹

파일과 폴더 — 모든 것의 기본文件与文件夹 —— 一切的基础

컴퓨터에 저장되는 모든 것은 파일이에요. 사진 한 장도 파일, 우리가 만들 웹앱도 파일이죠. 폴더는 그 파일들을 담는 상자예요. 우리는 내 프로젝트 폴더 하나를 만들고, 그 안에 파일들을 모아 작업할 거예요.存在电脑里的一切都是 文件。一张照片是文件，我们要做的网页应用也是文件。文件夹 就是装这些文件的盒子。我们会建 一个我的项目文件夹，把文件都放进去来做。

📄 파일 (File)文件 (File)

내용이 담긴 하나하나. 이름 뒤 확장자(.html 등)가 종류를 말해줘요.装着内容的一个个东西。名字后的 后缀(如 .html)表示类型。

📁 폴더 (Folder)文件夹 (Folder)

파일을 담는 상자. 폴더 안에 폴더도 들어가요.装文件的盒子。文件夹里还能放文件夹。

💡 확장자가 종류를 정해요后缀决定类型

index.html은 웹페이지, style.css는 꾸미기, photo.png는 그림. 파일 이름 끝의 점(.) 뒤 글자를 보면 그 파일이 뭔지 알 수 있어요.index.html 是网页，style.css 是样式，photo.png 是图片。看文件名末尾点(.)后面的字母，就知道它是什么。

기초 — 인터넷과 웹基础 — 互联网与网页

인터넷 · 웹 · 웹사이트 · 웹앱互联网 · 网页 · 网站 · 网页应用

비슷해 보이지만 조금씩 달라요. 한 번 정리하고 가면 헷갈리지 않아요.看起来像，其实各有不同。理一遍就不会搞混。

  • 인터넷 — 전 세계 컴퓨터를 잇는 거대한 도로망. 그 위로 정보가 오가요.互联网 —— 连接全世界电脑的巨大 公路网，信息在上面来往。
  • 웹(Web) — 그 도로 위에서 웹페이지들이 링크로 서로 연결된 세계. 우리가 브라우저로 보는 그것.网(Web) —— 那些公路上 网页用链接互相连接 的世界，就是我们用浏览器看的东西。
  • 웹사이트 — 하나의 주제로 묶인 웹페이지 모음 (예: 학교 홈페이지).网站 —— 围绕一个主题的 网页合集(比如学校主页)。
  • 웹앱 — 그냥 보기만 하는 게 아니라 버튼을 누르면 반응하고 뭔가를 해주는 웹사이트. 우리가 만들 것!网页应用 —— 不只是看，而是 点按钮会反应、会帮你做事 的网站。就是我们要做的!
🌐 브라우저가 웹을 ‘열어주는 창’浏览器是「打开网页的窗口」

크롬·사파리·엣지 같은 브라우저가 웹페이지 파일(HTML)을 받아 사람이 보는 화면으로 그려줘요. 우리가 만든 index.html도 브라우저로 열면 바로 화면이 돼요.Chrome·Safari·Edge 这些 浏览器 接收网页文件(HTML)，把它画成人能看的画面。 我们做的 index.html 用浏览器打开就直接变成画面。

기초 — 제품基础 — 产品

‘제품(product)’을 만든다는 것做一个「产品(product)」

우리는 그냥 코드를 짜는 게 아니라 ‘제품’을 만들어요. 제품이란 누군가의 문제를 해결해 주거나, 즐거움을 주는 것이에요. 게임도 제품, 할 일 목록 앱도 제품이죠.我们不只是写代码，而是 做「产品」。 产品就是 能解决某人的问题、或带来乐趣的东西。 游戏是产品，待办清单应用也是产品。

그래서 만들기 전에 항상 “누가, 무엇을, 왜 쓰지?”를 먼저 생각해요. 이 질문에 답을 적어두는 문서가 뒤에 나올 기획서(prd.md)예요. 코드보다 이 생각이 먼저예요.所以动手前总要先想 「谁、用什么、为什么用?」 把答案写下来的文档，就是后面会讲的 企划书(prd.md)。比起代码，这个想法在先。

개념 — 코딩概念 — 编程

코드 · 코딩 · 바이브 코딩代码 · 编程 · Vibe 编程

컴퓨터는 빠르지만 스스로 알아서 하지 못해요. “이걸 이렇게, 그다음 저렇게”를 순서대로 정확히 적어줘야 하죠. 그 적은 것이 코드, 코드를 쓰는 일이 코딩이에요.电脑很快，但 不会自己判断。 要 按顺序、精确地 写清「先这样，再那样」。写出来的就是 代码，写代码这件事就是 编程。

핵심核心

‘Vibe(바이브)’는 ‘분위기·느낌’이라는 뜻 — 딱딱한 문법 대신 “이런 느낌으로 만들어줘” 하고 대화하듯 만든다고 해서, AI 연구자 안드레이 카파시가 ‘바이브코딩’이라 불렀어요.「Vibe」是「氛围·感觉」的意思 —— 不抠语法，像聊天一样说「做成这种感觉」，所以 AI 研究者 Andrej Karpathy 把它叫「Vibe Coding」。

개념 — 중요한 2가지概念 — 两个重点

바이브코딩엔 이 2가지가 중요해요Vibe 编程，这两点最重要

AI가 코드를 대신 써주니, 남는 건 두 가지 — 만들 대상을 내가 알고, 그걸 잘 설명하는 거예요.AI 帮你写代码，剩下两点 —— 先懂 要做的东西，再 说清楚。

  • 내가 그 대상을 알기 — 만들려는 것 자체(내용·주제)를 알아야 무엇을 만들지 그려져요.先懂那个主题 —— 懂要做的东西本身(内容·主题)，才知道要做什么。
  • 구체적으로 잘 설명하기 — 아는 걸 또렷하게 말할수록 원하는 결과가 나와요.说得具体清楚 —— 把懂的东西讲明白，越接近想要的结果。

지금은 이 2가지만 하면 돼요. 코드 원리를 깊이 파고드는 건 전문가·개발자로 나아갈 때의 이야기고요. AI는 똑똑한 조수, 방향을 정하는 건 나예요.现在做好 这两点 就行。深挖代码原理，是 走向专家·开发者 时的事。AI 是聪明助手，定方向的是 我。

무엇을 만들지 정하고(PRD), 어떻게 시킬지(프롬프트·컨텍스트)를 배워요. 바이브 코딩의 진짜 실력이 여기 있어요.先定「做什么」(PRD)，再学「怎么让 AI 做」(提示词·上下文)。Vibe 编程真正的功力 就在这里。
개념 — 순서概念 — 顺序

제품 만드는 순서做产品的顺序

전문가들은 아이디어를 바로 코드로 만들지 않아요. 밑그림 → 뼈대 → 색 입힌 화면 → 눌러지는 시제품 → 진짜 도는 제품 순으로, 단계를 밟으며 위험을 줄여요. 용어가 낯설어도 흐름만 느껴 보세요.高手不会把点子 直接写成代码。而是按 草图 → 骨架 → 上色画面 → 可点样品 → 真正能跑的产品，一步步降低风险。词生疏没关系，先感受流程。

  • PRD — Product Requirements Document. 무엇을·왜 만들지 적는 기획서 (제일 먼저!)PRD — 产品需求文档。写清 做什么·为什么 的规划书(最先做!)
  • Wireframe — 색·꾸밈 없이 어디에 무엇이 있는지 뼈대만.Wireframe(线框图) — 无颜色装饰，只画 什么在哪 的骨架。
  • Mockup — 색·글꼴까지 입힌 실제로 보일 모습 (아직 안 눌림).Mockup(视觉稿) — 加上颜色·字体的 真实外观(还不能点)。
  • Prototype — 버튼이 눌러지고 화면이 넘어가는 시제품.Prototype(原型) — 按钮 能点、画面能跳转 的样品。
  • MVP — Minimum Viable Product. 핵심 하나가 진짜로 도는 최소 제품.MVP — 最小可行产品。核心功能真的能跑 的最小产品。
💡 우리의 압축 버전 & 왜 MVP부터?我们的压缩版 & 为何从 MVP?

우리는 PRD.md 작성 → Interactive Mockup.html 생성 → MVP 구현 세 단계로 압축해요. 완벽을 노리면 못 끝내요 — 핵심 하나가 도는 것을 먼저 만들고 거기서 키웁니다.我们压成三步: 写 PRD.md → 生成 Interactive Mockup.html → 实现 MVP。追求完美就做不完 —— 先做出 一个核心能跑的，再往上加。

핵심 — prd.md核心 — prd.md

prd.md — 왜, 무엇을 쓰나为什么、写什么

prd.md는 두 사람을 위한 문서예요 — 생각을 정리하는 나, 그리고 그걸 읽고 만드는 AI. 코드보다 이게 먼저예요.prd.md 是给 两个人 的文档 —— 梳理想法的 我，和读它来做的 AI。它比代码更先。

🧠 ① 나를 위해① 为了我

무엇을 만들지 스스로 설계·정리. 머릿속이 또렷해져요.把要做的 自己梳理清楚，思路更明确。

🤖 ② AI를 위해② 为了 AI

AI가 읽을 자료. “prd.md 참고해줘”라고 하면 이걸 보고 만들어요.给 AI 读的资料。说「参考 prd.md」，它就照这个做。

정답 형식은 없어요. 보통 이런 항목을 위→아래로 적고, 각 줄이 AI에게 주는 지시가 돼요.没有标准格式。通常从上到下写这些项，每行都是给 AI 的指示。

  • ① 한 줄 소개 — 뭘 만드는지 한마디로① 一句话介绍 — 一句话说清做什么
  • ② 누가 볼까·즐길까 — 대상 (화면·말투가 정해짐)② 给谁看·谁玩 — 对象(决定画面·语气)
  • ③ 화면 구성 — 위→아래로 뭐가 보이나③ 画面结构 — 从上到下有什么
  • ④ 핵심 기능·규칙 — 꼭 되는 동작 1~3개④ 核心功能·规则 — 必须能用的 1~3 个
  • ⑤ 담을 내용 — 실제 텍스트·데이터⑤ 要放的内容 — 真实文字·数据

개념 — 컨텍스트概念 — 上下文

컨텍스트 = AI가 지금 아는 정보上下文 = AI 此刻知道的信息

AI는 내 머릿속을 못 봐요. 내가 준 정보(창)만큼만 봅니다. 이 창의 크기를 컨텍스트 창이라고 해요. 물컵처럼 담을 수 있는 양에 한계가 있어요.AI 看不到我脑子里。它只看得到 我给的信息(窗口)。这个窗口的大小叫 上下文窗口。像水杯一样 能装的量有限。

  • 요즘 AI의 창은 대략 책 한 권 분량(예: 20만 토큰)까지 커졌어요.如今 AI 的窗口大到 约一本书的量(例如 20 万 tokens)。
  • 하지만 무한하지 않아요 — 대화가 길어지면 창이 꽉 차요.但 不是无限的 — 对话变长，窗口会被塞满。
  • 창이 차면 앞 내용을 잊거나 흐릿하게 기억해요.塞满后会 忘掉或记不清 前面的内容。
🔎 더 깊이 — ‘토큰’이 뭐예요?深入了解 — 「token」是什么?

AI는 글을 ‘토큰’이라는 작은 조각으로 잘라서 읽어요. 대략 한글은 한두 글자, 영어는 한 단어가 1토큰쯤이에요. “20만 토큰”은 대략 두꺼운 책 한 권 분량이고요. 대화·코드·파일을 넣을수록 토큰이 쌓여 창이 차요. 그래서 필요한 것만 넣는 게 요령이에요.AI 把文字切成叫 「token」 的小块来读。大致中文一两个字、英文一个词约 1 个 token。「20 万 token」大约是 一本厚书。对话·代码·文件放得越多，token 越堆越满。所以诀窍是 只放需要的。

💡 그래서所以

큰 걸 만들수록 “무엇을 기억시킬지”를 내가 관리해야 해요. 이게 컨텍스트 엔지니어링이에요.做得越大，越要 由我来管理「让它记住什么」。这就是 上下文工程。

⚠️ 창이 차가는 신호 — 자동 압축窗口快满的信号 — 自动压缩

대화가 길어지면 OpenCode가 이전 내용을 자동으로 요약(압축)해요 — 이때 세부가 흐려질 수 있어요. AI가 아까 정한 걸 잊은 것 같으면 바로 그 신호예요. 그래서 큰 기능 하나가 끝날 때마다 아래 context.md를 갱신해두면 안전해요.对话变长时，OpenCode 会 自动摘要(压缩) 前面的内容 —— 这时 细节会变模糊。AI 好像忘了刚定的事,就是这个信号。所以 每做完一个大功能,就更新下面的 context.md 更保险。

AI는 ‘글자’만 읽어요 — 그림 대신 말로AI 只读「文字」— 用说的代替图

우리가 쓰는 무료·가벼운 모델은 글자는 아주 잘 읽지만 그림(스크린샷·사진)은 못 봐요. 눈은 없고 귀만 있는 친구라고 생각하면 돼요 — 그래서 화면이 이상하면 사진 말고 ‘말’로 알려줘요.我们用的免费·轻量模型 很会读文字，但看不懂图(截图·照片)。把它想成 没有眼睛、只有耳朵的朋友 —— 所以画面不对时，别发图，用「说」的 告诉它。

🔎 더 깊이 — 커밋·푸시가 무슨 뜻?深入了解 — commit·push 是什么意思?

커밋(commit)은 “지금까지 바꾼 걸 하나의 세이브로 묶어 저장”이에요. 메시지(“버튼 추가”)를 붙여 나중에 뭘 했는지 알아보기 쉽게 해요. 푸시(push)는 그 세이브들을 내 컴퓨터에서 GitHub(인터넷)로 올려보내는 것. 그래서 흐름은 항상 스테이징(담기) → 커밋(저장) → 푸시(올리기) 순서예요.提交(commit) 是「把目前的改动打成一个存档保存」，配上消息(「加按钮」)方便以后知道做了什么。推送(push) 是把这些存档 从我的电脑传到 GitHub(互联网)。所以流程总是 暂存(装) → 提交(存) → 推送(传)。

🔎 더 깊이 — ‘harness’의 어원深入 — harness 的词源

원래 군장비·갑옷(장비)을 뜻하던 옛 프랑스어 harneis에서 왔어요 (더 거슬러 올라가면 ‘군대 보급품’). 나중에 말에 채우는 마구로 뜻이 굳었고요. 공통점은 “무언가를 부리려고 채우는 장비” — AI 하네스도 모델을 부리려 채우는 장비 세트예요.源自古法语 harneis(原意「军备·装备·盔甲」，更早可溯到「军队补给」)，后来固定为 「马具」。共同点是 「为了驾驭某物而套上的装备」 —— AI 的 harness 也是 为驾驭模型而套上的整套装备。

코드 실행代码运行

코드는 어떻게 ‘실행’될까?代码是怎么「运行」的?

우리가 쓴 코드는 사람이 읽는 글자예요. 컴퓨터(CPU)는 기계어(0과 1)만 알아들어서, 중간에 번역이 필요해요. 번역 방식은 크게 두 가지예요.我们写的代码是 给人读的文字。电脑(CPU)只懂 机器语言(0 和 1)，所以中间需要 翻译。翻译方式大体两种。

📦 컴파일编译

미리 통째로 번역해 실행파일로. 빠름. (C — 자바 — Swift)提前整体翻译 成可执行文件。快。(C — Java — Swift)

🎤 인터프리터解释器

한 줄씩 즉석 번역하며 실행. 유연. (파이썬 — JS)逐行即时翻译 并运行。灵活。(Python — JS)
