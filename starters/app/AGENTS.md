# 프로젝트 안내 (AI 에이전트용)

> 이 파일은 **OpenCode 같은 AI 코딩 도구가 자동으로 읽는** 안내서예요.
> (자동으로 안 읽히면 사용자가 "AGENTS.md 먼저 읽고 지켜줘"라고 말하면 돼요.)

**AI에게:** 사용자는 **코딩을 처음 배우는 중·고등학생**이야. 아래를 읽고 지켜줘.

## 이 프로젝트가 뭐야?
- **바이브코딩 특강**에서 만드는 **나만의 웹앱**(소개/서비스 페이지 등)이야.
- 목표: `prd.md`의 기획대로 **`index.html`을 시작점**으로 웹앱을 만들고 → **GitHub Pages**로 배포하기.

## 준비된 것 (이 폴더 안)
- `prd.md` — **기획서**. 무엇을 만들지 여기 있어. **항상 먼저 읽고 참고해.**
- `design.md` — 색·폰트·스타일 가이드. 이걸 따라 예쁘게 만들어.
- `context.md` — 지금까지 정한 것·다음에 할 일 메모.
- `index.html` — 아직 **빈 뼈대**. 여기에 만들면 돼.
- `assets/icons/` — 자주 쓰는 SVG 아이콘. `assets/RESOURCES.md` — 무료 리소스 출처.

## 규칙 (꼭 지켜)
1. **index.html 중심으로** — 작을 땐 HTML·CSS·JS를 `index.html` 하나에 두면 편해. 커지면 `style.css`·`script.js`로 나눠도 OK(상대경로로 연결). 라이브러리는 꼭 필요할 때만 CDN.
2. **상대경로만** — `assets/icon.svg` 처럼. `/`로 시작하지 마 (GitHub Pages에서 깨져).
3. **데이터는 localStorage** — 서버·DB 없이 브라우저에 저장. (여러 기기·사람과 **공유**하거나 **로그인**이 필요해지면 그때 **Firebase** — 수업 자료 6장 참고)
4. **초보용 한국어 주석**을 충분히 달아줘.
5. **한 번에 하나씩** — 큰 걸 한꺼번에 말고, 작은 단위로 만들고 확인하며 진행.
6. **애매하면 먼저 질문** — 기획이 부족하면 만들기 전에 사용자에게 물어봐 채워.

## 일하는 순서
1. `prd.md`(+`design.md`)를 읽어. 부족하면 사용자에게 **질문**해서 채우도록 도와줘.
2. **바로 코드 X — 계획 먼저.** 만들(또는 고칠) 내용을 **`plan.md`(또는 `context.md`)에 단계별 개발 계획**으로 적고, 사용자에게 보여줘 **검토·수정을 받은 뒤** 진행해. 각 단계는 작게(한 번에 하나씩 확인 가능).
3. 계획대로 **한 단계씩** 만들고 확인 (화면 뼈대 → 동작 → 꾸미기).
4. 바뀐 건 **완성된 전체 코드**로 보여줘.
5. 마무리하면 `plan.md`의 끝난 단계에 ✓ + `context.md`의 "다음에 할 일"을 갱신하자고 알려줘.

> ⚠️ **수정·기능 추가도 똑같이** — "바로 고쳐줘"가 와도, 먼저 **어디를 왜 바꿀지 계획**을 말하고 확인받은 뒤 실행해.
