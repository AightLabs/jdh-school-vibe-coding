# 📄 웹앱 기획서 (PRD)

> **PRD가 뭐예요?** Product Requirements Document — "무엇을·누구에게·어떻게" 만들지 적어두는 **기획 문서**예요.
> **두 가지 목적:** ① 내 머릿속 생각을 **스스로 정리**하고, ② **AI(OpenCode)가 읽을 자료**가 됩니다.
> OpenCode에게 *"이 폴더의 prd.md 참고해서 만들어줘"* 라고 하면 아래 내용을 읽고 그대로 만들어줘요.
> **쓰는 법:** 괄호 `( )` 안 설명을 지우고 내 내용으로 채우세요. **구체적으로** 적을수록 결과가 좋아져요.

---

## ① 한 줄 소개
> 이 앱이 한마디로 뭔지. (무엇을 하는 페이지인가?)
( 예: 나를 소개하고 좋아하는 것들을 모아둔 페이지 )

## ② 누가 볼까 (대상)
> 누가 이걸 보나? 대상을 알면 화면·말투가 정해져요.
( 예: 내 친구들, 처음 만나는 사람 )

## ③ 화면 구성 (위 → 아래 순서)
> 페이지를 위에서 아래로 훑으면 뭐가 있나. 순서대로 적어요.
( 예: 상단바 → 인사(큰 이름·사진) → 좋아하는 것 3가지 → 응원 버튼 → 링크 → 푸터 )

## ④ 핵심 기능
> "이건 꼭 돼야 해" 하는 동작 1~3개. (버튼을 누르면 무엇이 일어나나?)
( 예: 응원 버튼 → 숫자 +1 · 다크모드 전환 · 링크 버튼으로 이동 )

## ⑤ 담을 내용 (진짜 텍스트)
> AI가 화면에 넣을 **실제 문구**. 여기에 내 진짜 내용을 적어요.
- 이름:
- 한 줄 소개:
- 좋아하는 것 3가지:
- SNS/링크:

## ⑥ 분위기 · 색 · 글꼴
> 어떤 느낌? 색·모서리·글씨 크기 등.
( 예: 밝고 깔끔한 흰 배경 + 보라 포인트, 둥근 모서리, 큰 글씨 )

## ✅ 다 채운 예시 (이렇게 채우면 돼요)

- **① 한 줄 소개**: 우리 강아지 '콩이'를 자랑하는 팬 페이지
- **② 누가 볼까**: 반려동물을 좋아하는 친구들
- **③ 화면 구성**: 상단바(콩이네) → 큰 사진 + 이름 → 콩이 소개 3줄 → `간식 주기 ♥` 버튼(누르면 하트 수 증가) → 사진 갤러리 → 푸터
- **④ 핵심 기능**: ① 간식 버튼 누르면 하트 +1  ② 사진 클릭하면 크게 보기  ③ 다크모드 토글
- **⑤ 담을 내용**: 이름=콩이 / 소개="3살 말티즈, 산책과 간식을 사랑함" / 갤러리=사진 4장
- **⑥ 분위기**: 파스텔 노랑 배경, 둥근 카드, 큼직하고 귀여운 제목

## 💡 ① 최소한 채웠으면 → AI와 함께 다듬기
완벽하게 채우지 않아도 돼요. 최소한만 적었으면 OpenCode에 **검토를 받아요**:
```
이 폴더의 prd.md를 검토해줘. 빠진 항목이나 더 구체적으로 적으면 좋을 부분,
이런 앱이면 넣으면 좋을 기능을 나에게 하나씩 질문해줘.
```
> AI의 질문에 답하며 채워 나가면 prd.md가 훨씬 탄탄해져요. (내가 놓친 것도 AI가 챙겨줘요)

## 🚀 ② 어느 정도 됐으면 → 만들기
```
이 폴더의 prd.md를 참고해서, index.html 하나에 HTML/CSS/JS로 만들어줘.
초보용 한국어 주석을 달고, 완성된 전체 코드를 보여줘.
```
> 💡 흐름이 헷갈리면 먼저 *"눌러지는 mockup.html로 만들어줘"* 로 목업을 확인한 뒤 위 명령으로 완성해도 좋아요. (수업 자료 4장)
> 한 번에 다 말고 **하나씩**: 화면 뼈대 → 버튼 동작 → 색·꾸미기 순서로.
