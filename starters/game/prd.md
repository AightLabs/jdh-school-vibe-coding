# 🎮 게임 기획서 (PRD)

> 🎯 **목표:** 설치 없이 **브라우저에서 도는 웹 게임**을 `index.html` 하나로 만들어 → **GitHub Pages로 배포**(인터넷 주소로 공유)!
>
> **PRD가 뭐예요?** Product Requirements Document — "무엇을·어떻게" 만들지 적어두는 **기획 문서**예요.
> **두 가지 목적:** ① 내 게임 아이디어를 **스스로 정리**하고, ② **AI(OpenCode)가 읽을 자료**가 됩니다.
> OpenCode에게 *"이 폴더의 prd.md 참고해서 만들어줘"* 라고 하면 아래 내용을 읽고 그대로 만들어줘요.
> **쓰는 법:** 괄호 `( )` 안 설명을 지우고 내 게임 내용으로 채우세요. **구체적으로** 적을수록 결과가 좋아져요.

---

## ① 한 줄 소개
> 이 게임이 한마디로 뭔지.
( 예: 떨어지는 별을 받아 점수를 올리는 웹게임 )

## ② 누가 즐길까 (대상)
( 예: 쉬는 시간에 심심한 친구들 )

## ③ 장르 · 핵심 재미
> 무슨 장르이고, 뭐가 재미있나.
( 예: 피하기/받기 게임 · 아슬아슬하게 받아내는 손맛 )

## ④ 조작 방법
> 어떻게 조작하나. (키보드? 터치? 마우스?)
( 예: ← → 방향키 또는 화면 좌우 터치로 이동 )

## ⑤ 규칙 · 승패
> 점수·목숨·게임오버 조건.
( 예: 별 받으면 +1점 · 폭탄 닿으면 게임오버 · 30초 동안 최고 점수 도전 )

## ⑥ 화면에 보이는 것
> 게임 화면에 떠 있어야 하는 것들.
( 예: 점수, 남은 시간, 플레이어, 떨어지는 별·폭탄, 게임오버 화면 )

## ⑦ 난이도 변화
( 예: 시간이 지날수록 별이 점점 빨리 떨어짐 )

## ⑧ 분위기 · 그래픽 · 사운드
> 배경·색·효과음 느낌. (에셋 폴더의 이미지·소리를 쓸 수 있어요)
( 예: 밤하늘 배경, 반짝이는 별, 받을 때 '딩동' 효과음 )

---

## ✅ 다 채운 예시 (이렇게 채우면 돼요)

- **① 한 줄 소개**: 하늘에서 떨어지는 사과를 바구니로 받는 게임
- **② 누가 즐길까**: 짧게 즐길 거리를 찾는 친구들
- **③ 장르·재미**: 받기 게임 · 점점 빨라져서 손에 땀나는 재미
- **④ 조작**: ← → 방향키로 바구니 좌우 이동 (모바일은 터치)
- **⑤ 규칙·승패**: 사과 +1점 / 썩은 사과 닿으면 목숨 -1 / 목숨 3개 다 잃으면 게임오버
- **⑥ 화면 요소**: 점수, 남은 목숨(♥×3), 바구니, 떨어지는 사과·썩은 사과, 게임오버 화면
- **⑦ 난이도**: 10점마다 떨어지는 속도 +10%
- **⑧ 분위기**: 파란 하늘 + 초록 잔디 배경, 통통 튀는 효과, 받을 때 '팟!' 소리

---

## 💡 ① 최소한 채웠으면 → AI와 함께 다듬기
완벽하게 채우지 않아도 돼요. 최소한만 적었으면 OpenCode에 **검토를 받아요**:
```
이 폴더의 prd.md를 검토해줘. 빠진 항목이나 더 구체적으로 적으면 좋을 부분,
이런 게임이면 넣으면 좋을 요소를 나에게 하나씩 질문해줘.
```
> AI의 질문에 답하며 채워 나가면 prd.md가 훨씬 탄탄해져요. (내가 놓친 것도 AI가 챙겨줘요)

## 🚀 ② 어느 정도 됐으면 → 만들기
```
이 폴더의 prd.md를 참고해서, index.html 하나에 웹게임을 만들어줘.
(Phaser 3를 CDN으로 써도 좋아). assets 폴더의 이미지·소리를 쓰고,
초보용 한국어 주석을 달고, 완성된 전체 코드를 보여줘.
```
> 한 번에 다 말고 **하나씩**: 플레이어 이동 → 떨어지는 것 → 점수 → 게임오버 순서로.

## 🌱 확장 아이디어
- 목숨(하트) 시스템 · 최고 점수 저장 · 아이템(금별 +5)
- 배경음악·효과음 · 파티클(반짝이) · 난이도 단계
