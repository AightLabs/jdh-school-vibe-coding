# 프로젝트 안내 (AI 에이전트용)

> OpenCode 같은 AI 코딩 도구가 **자동으로 읽는** 안내서예요.
> (자동으로 안 읽히면 "AGENTS.md 먼저 읽고 지켜줘"라고 말하면 돼요.)

**AI에게:** 사용자는 **코딩 처음 배우는 중·고등학생**이야. 아래를 지켜줘.

## 이 프로젝트 (목표를 꼭 기억해)
- 설치 없이 **브라우저에서 도는 웹 게임(웹앱)**이야 — HTML/CSS/JS(원하면 Phaser 3)로 만들어. 네이티브·설치형 게임이 아니야.
- **최종 목표: `prd.md` 기획대로 `index.html` 하나에 웹 게임을 만들어 → `GitHub Pages`로 배포**(인터넷 주소로 친구와 공유).
- 그러니 항상 **웹(브라우저)에서 그냥 열면 돌아가는** 형태로 만들어줘. 파일은 `index.html`을 중심으로(작을 땐 한 파일이 편함), 커지면 `script.js` 등으로 나눠도 OK.

## 준비된 것 (이 폴더 안)
- `prd.md` — **게임 기획서**. 먼저 읽고 참고해.
- `design.md` — 색·폰트·게임 느낌(효과음·파티클 등) 가이드. 이걸 따라 만들어.
- `context.md` — 정한 것·다음에 할 일.
- `index.html` — 아직 **빈 뼈대**.
- `assets/images/` — 스프라이트(SVG). `assets/audio/` — 소리(비어 있음, `RESOURCES.md` 참고).

## 규칙 (꼭 지켜)
1. **index.html 중심으로** — 작을 땐 HTML·JS를 `index.html` 하나에 두면 편해. 커지면 `script.js`로 나눠도 OK(상대경로로 연결).
2. **Phaser 3**를 쓰면 CDN으로 (`index.html` 상단 주석 참고). 안 써도 돼.
3. **상대경로만** — `assets/images/star.svg` (`/`로 시작 X).
4. **점수·최고점수 저장**은 localStorage. (여러 명 **순위표**나 **로그인**이 필요해지면 그때 **Firebase** — 수업 자료 6장 참고)
5. **초보용 한국어 주석**을 충분히.
6. **한 번에 하나씩** — 플레이어 이동 → 떨어지는 것 → 점수 → 게임오버 순서로.
7. **애매하면 먼저 질문**.

## 순서
1. `prd.md`를 읽고, 부족하면 **질문**해서 채우도록 도와줘.
2. 조금씩 만들고 확인.
3. 바뀐 건 **완성된 전체 코드**로 보여줘.
4. 마무리하면 `context.md` 갱신을 알려줘.
